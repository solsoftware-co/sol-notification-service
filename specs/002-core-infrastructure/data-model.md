## Entities

### 1. Client

A business customer of the notification service. One client may receive many notifications.

**Table**: `clients`

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `TEXT` | PRIMARY KEY | UUID generated by the caller (deterministic for seeding) |
| `name` | `TEXT` | NOT NULL | Display name (e.g., "Acme Corp") |
| `email` | `TEXT` | NOT NULL | Primary notification email address |
| `ga4_property_id` | `TEXT` | NULL | Google Analytics 4 property identifier (e.g., "properties/123456789") |
| `active` | `BOOLEAN` | NOT NULL, DEFAULT true | Whether this client receives notifications |
| `settings` | `JSONB` | NOT NULL, DEFAULT '{}' | Flexible key-value store for future customization |
| `created_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT NOW() | Record creation timestamp |

**Validation rules**:
- `id`: non-empty string
- `email`: non-empty, must contain `@`
- `name`: non-empty string
- `active`: must be boolean (not null)
- `settings`: valid JSON object (defaulted at DB level)

**State transitions**: `active` flag toggles between active and inactive. No formal state machine â€” toggled directly in the database.

**TypeScript representation** (`src/types/index.ts`):
```typescript
export interface ClientRow {
  id: string;
  name: string;
  email: string;
  ga4_property_id: string | null;
  active: boolean;
  settings: Record<string, unknown>;
  created_at: Date;
}
```

### 2. Notification Log

A record of a single notification delivery attempt. Table is created in this feature; active writes are deferred to a future feature.

**Table**: `notification_logs`

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `BIGSERIAL` | PRIMARY KEY | Auto-incrementing surrogate key |
| `client_id` | `TEXT` | NOT NULL, FK â†’ clients(id) | Which client was notified |
| `workflow` | `TEXT` | NOT NULL | Inngest function ID (e.g., `send-form-notification`) |
| `event_name` | `TEXT` | NOT NULL | Triggering Inngest event name (e.g., `form/submitted`) |
| `outcome` | `TEXT` | NOT NULL | Delivery result: `success`, `failed`, `skipped` |
| `created_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT NOW() | When the notification attempt occurred |

**TypeScript representation** (for future use):
```typescript
export interface NotificationLogRow {
  id: number;
  client_id: string;
  workflow: string;
  event_name: string;
  outcome: 'success' | 'failed' | 'skipped';
  created_at: Date;
}
```

### 3. Environment Configuration (runtime, not persisted)

The complete set of runtime settings derived from environment variables at service startup.

**TypeScript representation** (`src/types/index.ts`):
```typescript
export type AppEnv = 'development' | 'preview' | 'production';
export type EmailMode = 'mock' | 'test' | 'live';

export interface AppConfig {
  env: AppEnv;
  emailMode: EmailMode;
  testEmail: string | null;   // required when emailMode === 'test'
  resendApiKey: string | null; // required when emailMode === 'live'
  resendFrom: string;          // sender address, e.g. "Notifications <notify@domain.com>"
  databaseUrl: string;
}
```

**Derivation rules**:
```
VERCEL_ENV not set           â†’ env = 'development'
VERCEL_ENV = 'preview'       â†’ env = 'preview'
VERCEL_ENV = 'production'    â†’ env = 'production'

env = 'development' + no EMAIL_MODE override â†’ emailMode = 'mock'
env = 'preview'     + no EMAIL_MODE override â†’ emailMode = 'test'
env = 'production'  + no EMAIL_MODE override â†’ emailMode = 'live'
EMAIL_MODE = 'mock'|'test'|'live' (explicit)  â†’ override applied
```

### 4. Email Delivery Request (runtime value object, not persisted)

A named set of attributes describing a single outbound notification.

**TypeScript representation** (`src/types/index.ts`):
```typescript

  subject: string;
  html: string;

  originalTo: string;

  outcome: 'sent' | 'logged';
  resendId?: string;     // populated in live mode on success
}
```

## SQL DDL (used in `scripts/setup-db.ts`)

```sql
CREATE TABLE IF NOT EXISTS clients (
  id             TEXT        PRIMARY KEY,
  name           TEXT        NOT NULL,
  email          TEXT        NOT NULL,
  ga4_property_id TEXT       NULL,
  active         BOOLEAN     NOT NULL DEFAULT TRUE,
  settings       JSONB       NOT NULL DEFAULT '{}',
  created_at     TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE IF NOT EXISTS notification_logs (
  id          BIGSERIAL   PRIMARY KEY,
  client_id   TEXT        NOT NULL REFERENCES clients(id),
  workflow    TEXT        NOT NULL,
  event_name  TEXT        NOT NULL,
  outcome     TEXT        NOT NULL,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```
